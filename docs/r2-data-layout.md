# R2 Data Layout

Arktrace uses Cloudflare R2 (S3-compatible) to distribute pre-built pipeline
artifacts so users can start the dashboard without running the pipeline
themselves.

A dedicated public bucket (`arktrace-public`) is used exclusively for public
OSS artifacts, so no sub-prefix is needed — all objects live at the bucket
root.

---

## Bucket layout

```
arktrace-public/                   ← dedicated public bucket (Cloudflare R2)
  latest                           ← plain text: "20260412T120000Z"
  20260412T120000Z.zip              ← latest generation (1 kept by default)
  gdelt.lance.zip                   ← shared GDELT Lance store (~1.2 GB)
                                      pushed separately; only updated when
                                      GDELT is re-ingested
  validation_metrics.json           ← scoring validation metrics (tiny)
```

Each **generation zip** contains artifacts for all five regions.  Users
extract only the region(s) they need (controlled by `--region` on pull):

| File in zip | Description |
|---|---|
| `{region}.duckdb` | AIS positions, vessel features, scoring results |
| `{region}_watchlist.parquet` | Ranked candidate watchlist |
| `{region}_causal_effects.parquet` | C3 causal sanction-response model output |
| `{region}_graph/` | Lance Graph ownership chains (nodes + edges) |
| `validation_metrics.json` | Aggregate precision/recall metrics |

## Data classification

### By generation method

| Artifact | Generated by | Reason |
|---|---|---|
| `{region}.duckdb` | **CI** | Built from seeded synthetic vessels + public data; no live AIS key required |
| `{region}_watchlist.parquet` | **CI** | Scoring pipeline output over the DuckDB above |
| `{region}_causal_effects.parquet` | **CI** | Causal model output over the DuckDB above |
| `{region}_graph/` | **CI** | Ownership graph built from vessel registry seeded from DuckDB |
| `validation_metrics.json` | **CI** | Computed from labeled watchlist; no external data required |
| `gdelt.lance` | **CI** | Built from GDELT public HTTP feed (no API key) |
| Live AIS data in DuckDB | **Local + upload** | Requires `AISSTREAM_API_KEY`; CI uses synthetic seed instead |

### By scope

| Artifact | Scope | Notes |
|---|---|---|
| `gdelt.lance` | **Common** | Single GDELT corpus queried by all regions |
| `validation_metrics.json` | **Common** | Aggregate metrics across all regions |
| `{region}.duckdb` | **Regional** | Separate per region |
| `{region}_watchlist.parquet` | **Regional** | |
| `{region}_causal_effects.parquet` | **Regional** | |
| `{region}_graph/` | **Regional** | Ownership chains differ by region vessel pool |

## Bucket size budget

Target: stay under **10 GB**.

| Item | Estimated size | Notes |
|---|---|---|
| Generation zip × 1 (all 5 regions) | 500 MB – 1.5 GB | DuckDB (synthetic) + graphs + parquets |
| `gdelt.lance.zip` | ~1.2 GB | Pushed separately; 1 copy only |
| `validation_metrics.json` | < 1 MB | |
| **Total** | **~2 – 3 GB** | Well within 10 GB |

**Key decisions that keep the bucket small:**

- **`--keep 1` (default)** — only the latest generation zip is retained.
  Git history provides rollback if needed; R2 is not a backup system.
  Pass `--keep N` explicitly to retain more generations.
- **Marine Cadastre is not cached in R2** — the archive is multi-GB per year
  and already available publicly from the source.  Download it directly when
  needed (see [Marine Cadastre](#marine-cadastre) below).
- **`gdelt.lance` is separate from the rotation** — at ~1.2 GB it would
  dominate the budget if duplicated once per generation.  It is a single
  object updated only when GDELT is re-ingested.
- **Intermediate pipeline artifacts are excluded** — `anomaly_scores.parquet`,
  `composite_scores.parquet`, `mpol_baseline.parquet`, `mpol_graph/`, and all
  evaluation/backtest files are never zipped or uploaded.

## CI workflow

The `data-publish` workflow (`.github/workflows/data-publish.yml`) runs the
full multi-region pipeline on public data and publishes the results.

**Steps:**

1. Run `scripts/run_public_backtest_batch.py` for all five regions
   (`--stream-duration 0 --seed-dummy`).
2. `sync_r2.py push --keep 1` — zips all region artifacts into one file,
   uploads to R2, and deletes any previous generation.
3. `sync_r2.py push-gdelt` — uploads `gdelt.lance.zip` only if it does not
   already exist (skipped on most runs; re-run with `--force` after GDELT
   re-ingestion).

Trigger: manual dispatch (`workflow_dispatch`) or automatically after
`Public Backtest Integration` succeeds on `main`.

## User pull options

Use `scripts/sync_r2.py` to pull data locally before running the dashboard.

```bash
# Pull one region (minimum download — recommended for most users)
uv run python scripts/sync_r2.py pull --region singapore

# Pull a different region
uv run python scripts/sync_r2.py pull --region japan

# Pull multiple regions at once
uv run python scripts/sync_r2.py pull --region singapore,japan

# Also pull the GDELT Lance store (~1.2 GB) for analyst brief / chat context
uv run python scripts/sync_r2.py pull-gdelt

# Show what generations exist in R2 and their sizes
uv run python scripts/sync_r2.py list
```

After pulling, start the dashboard:

```bash
bash scripts/run_app.sh --region singapore
# or, with Docker:
docker compose up
```

### Approximate download sizes

| Command | Approx. size |
|---|---|
| `pull --region <region>` | 100 – 300 MB |
| `pull --region all` | 500 MB – 1.5 GB |
| `pull-gdelt` | ~1.2 GB |

## Credentials

R2 credentials are read from environment variables (or `.env`):

| Variable | Description |
|---|---|
| `S3_BUCKET` | `arktrace-public` |
| `S3_ENDPOINT` | `https://<account_id>.r2.cloudflarestorage.com` |
| `AWS_ACCESS_KEY_ID` | R2 Access Key ID (write only — not needed for pull) |
| `AWS_SECRET_ACCESS_KEY` | R2 Secret Access Key (write only — not needed for pull) |
| `AWS_REGION` | `auto` |

These credentials are **only** used by `sync_r2.py` and `bootstrap.py` for
push/pull. The dashboard always reads from local disk — setting these
variables does not route app reads to R2 (see `USE_S3` in
[Deployment](deployment.md) for the opt-in S3 serving mode).

**Read-only pulls require no credentials.** The `arktrace-public` bucket is
fully public (Cloudflare R2 → Settings → Public Access). Set only
`S3_BUCKET` and `S3_ENDPOINT` in `.env` to pull without a key.

### CI secrets

CI needs write credentials to push artifacts. Configure these as repository
secrets:

| Secret | Maps to | Value |
|---|---|---|
| `R2_BUCKET` | `S3_BUCKET` | `arktrace-public` |
| `R2_ENDPOINT` | `S3_ENDPOINT` | `https://<account_id>.r2.cloudflarestorage.com` |
| `R2_ACCESS_KEY_ID` | `AWS_ACCESS_KEY_ID` | R2 write token key ID |
| `R2_SECRET_ACCESS_KEY` | `AWS_SECRET_ACCESS_KEY` | R2 write token secret |

## Marine Cadastre

Marine Cadastre annual AIS track archives are multi-GB per year and available
publicly from `marinecadastre.gov`.  They are **not** cached in R2 — download
directly from the source when you need to run the full pipeline with historical
US coastal AIS data:

```bash
uv run python -m src.ingest.marine_cadastre --year 2024 \
    --output data/raw/marine_cadastre
```

The pipeline skips this step automatically when the local archive is absent
(`--stream-duration 0 --seed-dummy` for CI runs).

## Proprietary data

Live AIS data collected via AISstream.io, AISHub, or Global Fishing Watch
(GFW) is **not** published to R2.  If you have collected your own live AIS
data and want to share it within your team:

1. Push your regional DuckDB (which contains the live positions) to a
   **private** R2 bucket, separate from `arktrace-public`.
2. Share credentials only with authorised team members.
3. Do not push to `arktrace-public` — that bucket is reserved for OSS
   artifacts that can be freely distributed.
